DESIGN PRINCIPLES â€” WHY YOU CAN TRUST THE RECORD
01

Append-Only Writes

The audit log is a permanent record â€” data is only ever added, never changed or deleted. Like a physical logbook: you can write a new line, but you can't erase what's already there. This guarantees the record reflects exactly what happened.

02

State is Always Derived

The "current picture" of an incident is always computed by replaying the audit log from scratch. You can delete the state files and regenerate them â€” you'll get the exact same result. The log is the truth; everything else is just a convenient view of it.

03

No Hidden State

The notebooks and tools don't hold any critical data in memory. Everything important lives in files. This means you can restart the tool, re-run any command, and get the same result â€” nothing is lost if the software crashes mid-incident.

04

Human-Gated AI

The AI advisory layer is intentionally separated from the core system â€” it can only communicate through a dedicated exchange folder. A human analyst reviews every AI suggestion before it enters the official record. AI advises; humans decide.

05

Tamper-Evident

Every log file has a cryptographic fingerprint (SHA-256 hash) displayed on the dashboard. If anyone modifies the record after the fact, the fingerprint changes â€” making tampering immediately detectable. Built for regulatory and legal defensibility.

06

Plain-Text & Portable

All log files use standard text formats that work with common tools â€” no proprietary database or special software required to inspect the record. One entry per line, readable by any analyst on any system. Easy to search, filter, compare, and archive.
